1. NXP S32K566 — Device Overview
The S32K566 belongs to NXP's new S32K5 family, announced at Embedded World in March 2025. It is
the automotive industry's first microcontroller built on a 16 nm FinFET process node with embedded
Magnetic RAM (MRAM). The family targets zonal and domain controllers, battery management systems
(BMS), on-board chargers (OBC), traction inverters, and high-performance ECU consolidation use
cases in Software-Defined Vehicle (SDV) architectures.

The Cortex-M7 cores support split-lock architecture, meaning any pair can run independently (ASIL B,
higher throughput) or in lockstep (ASIL D, maximum safety). XRDC (Extended Resource Domain
Controller) enforces hardware-level isolation between domains, which is essential for mixed-criticality
ECU consolidation.

#include "S32K5FlexCanTransceiver.h"
// FlexCAN register offsets (relative to base address)
static constexpr uint32_t FLEXCAN_MCR
= 0x00;
static constexpr uint32_t FLEXCAN_CTRL1
= 0x04;
static constexpr uint32_t FLEXCAN_TIMER
= 0x08;
static constexpr uint32_t FLEXCAN_IMASK1
= 0x28;
static constexpr uint32_t FLEXCAN_IFLAG1
= 0x30;
static constexpr uint32_t FLEXCAN_MB_BASE = 0x80; // MB0 starts at offset 0x80
// Each MB = 16 bytes (CS word + ID word + 8 data bytes + padding)
#define CAN_REG(offset)

(*(_can_base + ((offset) >> 2)))

::can::ICanTransceiver::ErrorCode S32K5FlexCanTransceiver::init() {
configureClocks();
configurePins();
// 1. Request Freeze mode
CAN_REG(FLEXCAN_MCR) |= (1U << 25); // MDIS=0, FRZ=1
CAN_REG(FLEXCAN_MCR) |= (1U << 24); // HALT=1
while (!(CAN_REG(FLEXCAN_MCR) & (1U << 8))) {} // wait FRZACK
// 2. Set 500 kbps timing (PRESDIV, PSEG1, PSEG2, RJW)
CAN_REG(FLEXCAN_CTRL1) = BAUD_500K;
// 3. Configure Tx MB (MB0) and Rx MB (MB4)
configureMsgBuffer(TX_MB_IDX, /*isTx=*/true);
configureMsgBuffer(RX_MB_IDX, /*isTx=*/false, 0x200); // accept 0x200
// 4. Enable Rx interrupt for MB4
CAN_REG(FLEXCAN_IMASK1) = (1U << RX_MB_IDX);
// 5. Exit freeze mode
CAN_REG(FLEXCAN_MCR) &= ~((1U << 25) | (1U << 24));
while (CAN_REG(FLEXCAN_MCR) & (1U << 8)) {} // wait FRZACK clear

}

return ErrorCode::CAN_ERR_OK;

::can::ICanTransceiver::ErrorCode
S32K5FlexCanTransceiver::write(::can::CANFrame const& frame) {
// MB word layout: [CS(0)] [ID(1)] [DATA_WORD0(2)] [DATA_WORD1(3)]
uint32_t* mb = reinterpret_cast<uint32_t*>(CAN0_BASE + FLEXCAN_MB_BASE + TX_MB_IDX*16);
mb[1] = (frame.getId() & 0x7FFU) << 18; // standard ID in bits 28:18
const uint8_t* d = frame.getPayload();
mb[2] = ((uint32_t)d[0]<<24)|((uint32_t)d[1]<<16)|((uint32_t)d[2]<<8)|d[3];
mb[3] = ((uint32_t)d[4]<<24)|((uint32_t)d[5]<<16)|((uint32_t)d[6]<<8)|d[7];
// CS word: CODE=0xC (Tx data frame), DLC=8
mb[0] = (0xCU << 24) | (frame.getPayloadLength() << 16);
}

return ErrorCode::CAN_ERR_OK;

void S32K5FlexCanTransceiver::handleRxIrq() {
uint32_t flags = CAN_REG(FLEXCAN_IFLAG1);
if (flags & (1U << RX_MB_IDX)) {
uint32_t* mb = reinterpret_cast<uint32_t*>(
CAN0_BASE + FLEXCAN_MB_BASE + RX_MB_IDX*16);
uint32_t cs = mb[0];
uint32_t id = (mb[1] >> 18) & 0x7FFU;
uint8_t dlc = (cs >> 16) & 0xFU;
::can::CANFrame frame;
frame.setId(id);
uint8_t data[8];
for (uint8_t i = 0; i < dlc; ++i) {
uint8_t word = i / 4;
uint8_t byte = 3 - (i % 4); // big-endian in MB
data[i] = (mb[2 + word] >> (byte * 8)) & 0xFF;
}
frame.set(data, dlc);
if (_listener) { _listener->canFrameReceived(frame); }

}

}

// Clear flag by writing 1
CAN_REG(FLEXCAN_IFLAG1) = (1U << RX_MB_IDX);
(void)mb[0]; // unlock MB (read CS to unlock)

extern "C" void FlexCAN0_ORed_MB_0_15_IRQHandler() {
// Assumes a global pointer or singleton to the transceiver instance
extern S32K5FlexCanTransceiver* g_canTransceiver;
if (g_canTransceiver) g_canTransceiver->handleRxIrq();
}

4. Running the Application on Synopsys VDK

announced the S32K5 VDK at Embedded World March 2025. Elektrobit, Green Hills Software, and
other ecosystem partners have already ported their AUTOSAR stacks to it. This section explains how to
use the VDK to run and verify the CAN application without hardware.

5. Common Issues & Troubleshooting
Symptom

Likely Cause & Fix

FlexCAN stays in freeze mode

Check that PCC (Peripheral Clock Configuration) has enabled
the CAN clock gate before writing MCR. Also verify PLL is
locked before setting CAN clock source.

No RX interrupt fires

Verify IMASK1 is set for the correct MB index. Confirm the
MB's CS CODE is 0x4 (empty, ready to receive), not 0x0
(inactive).

VDK GDB connection refused

Ensure the VDK process is fully started before connecting (look
for 'Listening on port 2345' in VDK console output).

Python can_stimulus.py gets no
reply

The CAN application may still be in init. Add a small sleep(0.5)
before sending. Verify loopback is OFF in params.yaml.

ELF loads but CPU hangs at reset

Check the linker script — the MRAM origin must match the
S32K566 boot address (typically 0x00400000 for code MRAM)
and the VDK params MRAM_size.

Wrong baud rate on UART console

The VDK LPUART model uses the host_port but does not
enforce baud rate — nc always works. If using a real terminal,
ensure baud_rate: 115200 in params.yaml matches your
LPUART init.
